# Правила внесения вклада

## Требования к pull request

Согласно условиям лицензии ЛЕВИ, при внесении любых изменений в функционал или добавлении новых возможностей, вы **обязаны** предлагать эти изменения через pull request в основной репозиторий.

## Процесс рассмотрения

1. Создайте fork репозитория
2. Внесите изменения в вашем fork
3. Откройте pull request в основной репозиторий
4. Опишите внесенные изменения и их назначение

## Важные замечания

- Автор оставляет за собой право отклонить любой pull request без объяснения причин
- Изменения должны соответствовать изначальной цели проекта — автоматизации создания учебных отчетов
- Все вносимые изменения должны сопровождаться соответствующей документацией
- При использовании кода, сгенерированного с помощью ЛЛМ, обязательно укажите это в комментариях

## Тестирование

Перед отправкой pull request убедитесь, что:
- Система сборки работает корректно
- Новые функции не нарушают существующую функциональность

Спасибо за ваш вклад в развитие проекта!
